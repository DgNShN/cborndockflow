# Security Policy

## Supported versions

Security fixes are applied to the active `main` branch.

## Reporting a vulnerability

If you find a security issue, please do not open a public issue with exploit details.

Instead:

1. Open a private security advisory on GitHub if available.
2. Or contact the repository maintainer through a private channel.

## What to include

- A clear description of the issue.
- The affected files or workflow.
- Steps to reproduce, if safe to share.
- Any suggested mitigation.

## Safe handling

Do not include secrets, access tokens, private keys, or user data in reports, screenshots, or commits.
